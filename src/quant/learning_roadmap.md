## 设计思路：为什么这么排

普通量化入门教材会把大量篇幅花在 MACD / KDJ / K 线形态上——那是**散户技术分析**的传统，在严肃量化中信噪比极低。这份路线图针对你的背景做了重新定位：

**你的真优势（应重点放大）**

| 算法能力 | 量化对应方向 |
|---|---|
| 特征工程 | 因子挖掘（Alpha Research） |
| 树模型 / 集成 | 横截面选股的主力模型 |
| 凸优化 | 投资组合优化（CVXPY） |
| 统计严谨性 | 防过拟合、风险归因 |
| 强化学习 | 仓位 / 执行优化（而非价格预测） |

**A 股机构量化的主战场（本路线图的核心）**

```
多因子模型 ──▶ 组合优化 ──▶ Alpha 对冲（市场中性）
   │              │              │
   └─ 挖 Alpha     └─ 配置权重    └─ 剥离 Beta，只赚 Alpha
```

这三件事是当前 A 股量化私募 / 公募量化的标准打法，也是个人研究者最有可能做出超额收益的方向。**传统技术指标（MA/MACD/RSI/KDJ 等）降级为附录速查**，需要时查阅即可。

**需要补足的短板**：市场交易机制、成本与约束、风险归因、金融时序的特殊统计性质。

## 阶段零：市场与交易机制

### 核心目标

建立正确认知：**交易规则不是背景知识，而是你模型里的硬约束**。T+1、涨跌停、成本都会直接进入策略逻辑和回测引擎。

### 0.1 必备规则（算法视角）

| 概念 | 规则 | 对模型的含义 |
|---|---|---|
| 交易时段 | 9:30–11:30, 13:00–15:00 | 数据采样窗口 |
| T+1 制度 | 当日买入次日才能卖出 | 时序动作的延迟约束，禁止日内换手 |
| 涨跌停 | 主板 ±10%，科创/创业 ±20%，北交所 ±30% | 动作可行性约束：涨跌停板**买不进 / 卖不出** |
| 最小变动价 | 0.01 元（股票） | 价格离散化 |
| 手 | 100 股为 1 手 | 下单量的整数约束 |
| 复权 | 前/后复权 | 因子计算前必须处理，否则有未来信息 |

### 0.2 交易成本（2026 现行）

```
买入单边 = 佣金（万 1~万 2.5，券商可谈）+ 过户费（万 0.1）        ≈ 万 1.1 ~ 万 2.6
卖出单边 = 佣金 + 过户费 + 印花税（0.5‰ = 万 5，2023.8.28 起减半）≈ 万 6.1 ~ 万 7.6
一个完整来回（买+卖）= 买入单边 + 卖出单边                       ≈ 万 7.2 ~ 万 10.2 ≈ 0.07% ~ 0.10%

注：印花税只在卖出单边收取；佣金最低可谈到万 0.85（取消「最低 5 元」后）。
若再叠加滑点（见下文，单边约万 1~万 3），来回总成本约 0.09% ~ 0.16%。
```

> ⚠️ 网上很多教程仍写印花税 1‰，那是 2023 年 8 月之前的旧值。

**为什么成本是量化第一性原理**：
- 中高频策略的收益常常**吃不掉成本**；
- 回测不计入佣金 / 滑点 / 冲击成本 = "纸上富贵"；
- 滑点：期望价 vs 实际成交价之差，流动性越差越大；大单还会产生**冲击成本**（market impact）。

### 0.3 订单类型与执行算法

| 订单类型 | 含义 |
|---|---|
| 限价单 | 指定价格挂单 |
| 市价单 | 以最优价立即成交（有滑点）|
| TWAP | 按时间均分大单，减小冲击 |
| VWAP | 按历史成交量分布拆单 |
| IS（Implementation Shortfall）| 平衡冲击成本与择时风险 |

> 实盘资金量上去后，**执行算法**（算法交易 / 母单拆分）本身就是一门学问，直接影响 Alpha 能落地多少。

### 🎯 阶段零项目

写一个**成本感知的成交模拟器**：输入"目标持仓 + 当日行情"，输出考虑涨跌停限制、佣金、过户费、印花税、固定滑点后的真实成交净值。

**✅ 完成标志**：
- [ ] 能默写 A 股 T+1 / 涨跌停 / 印花税
- [ ] 解释"为什么日内在 A 股现货上几乎不可行"
- [ ] 成本模拟器跑通并核对账目

## 阶段一：数据工程

### 核心目标

构建一个**无未来函数（no look-ahead bias）**的因子数据面板。数据工程做不好，后面所有 Alpha 都是假的。

### 1.1 数据源

| 数据源 | 类型 | 覆盖 | 成本 | 适用 |
|---|---|---|---|---|
| Tushare Pro | 接口 | A 股 / 港美股 / 期货 / 财务 | 积分制 | 个人研究主力 |
| AkShare | 接口 | 全市场聚合 | 免费 | 灵活但不稳定 |
| Baostock | 接口 | A 股历史 | 免费 | 历史量价/财务 |
| 聚宽 / 米筐 | 平台 | A 股全 | 免费+付费 | 在线研究/回测 |
| Wind / Choice / 同花顺 iFinD | 终端 | 全市场 | 昂贵 | 机构标配 |

### 1.2 数据类型

```
量价数据：开高低收量额（OHLCV），日线 / 分钟 / tick
财务数据：利润表、资产负债表、现金流量表（注意披露时点！）
基本面：行业分类、PE/PB/ROE、市值
另类数据：舆情、供应链、卫星、招聘、专利（进阶）
```

### 1.3 数据质量的核心坑（必踩必懂）

| 坑 | 说明 | 正确做法 |
|---|---|---|
| **未来函数** | 财务数据用了"未来才公布"的值 | 用 **point-in-time** 数据，按公告日对齐 |
| **复权方式** | 除权除息导致价格跳变 | 因子计算用**后复权**，展示用前复权 |
| **幸存者偏差** | 只回测现在还存在的股票，漏掉退市股 | 股票池必须包含已退市标的 |
| **停牌处理** | 停牌期间无数据 | 明确填充策略或剔除，记录在册 |
| **ST / 新股** | 上市初期、戴帽摘帽异常波动 | 通常剔除新股（如上市 < 60 日）和 ST |
| **时点对齐** | 不同频率数据合并错位 | 统一到交易日历，用 `as-of` join |

> point-in-time（PIT）是机构量化的基本要求：2024Q3 财报可能在 2025 年 4 月才披露，因子计算时必须用"当时已知"的值。Tushare 的 `fina_indicator` 等接口需要你自行处理披露日。

### 🎯 阶段一项目

构建一个**面板数据 DataFrame**（multi-index: date × ticker），包含后复权 OHLCV + 关键财务因子，且全部按时点对齐。写单测验证：任一历史日期上，取到的财报数据确实是当时已披露的。

**✅ 完成标志**：
- [ ] 能拉取并清洗 10 年以上全 A 股日线
- [ ] 财务因子按公告日对齐，无未来函数
- [ ] 退市股票保留在面板中

## 阶段二：因子研究方法论

### 核心目标

掌握量化研究员的**日常语言和工作流**：因子是什么、怎么评价、怎么处理、怎么合成。这是量化区别于"技术分析派"的分水岭。

### 2.1 因子的本质：横截面 vs 时序

```
时序策略：同一资产，看自己的过去预测未来（趋势跟踪）   ← 信噪比低
横截面策略：同一时刻，比较不同资产，预测相对排序（选股） ← A 股 Alpha 主流
```

> 对算法工程师：横截面选股本质上是一个**排序学习 / 分类的横截面问题**——这正是你的舒适区。绝大多数 A 股量化 Alpha 来自横截面，而非单资产时序预测。

### 2.2 因子来源

- **量价因子**：动量、反转、波动率、换手率、量价背离
- **财务因子**：价值（PE/PB）、质量（ROE/ROA）、成长（营收增速）
- **规模因子**：市值（小市值效应在 A 股长期显著）
- **风格因子**：低波、低换手
- **表达式因子库**：**WorldQuant Alpha101**（101 个公式化因子）、**qlib Alpha158**（现成基准，建议直接跑）

### 2.3 因子评价体系（必背）

| 指标 | 含义 | 经验值 |
|---|---|---|
| **IC** | 因子值与下期收益的相关系数 | 月度 |IC| > 0.03 即有用 |
| **Rank IC** | 用 Spearman 秩相关，抗极值 | 更常用 |
| **IR** | IC 均值 / IC 标准差 | > 0.5 优秀 |
| **ICIR** | 信息比率（同 IR，按 IC 序列）| 衡量因子稳定性 |
| **分组单调性** | 按因子分 5/10 组，收益是否单调 | 多空组价差要显著 |
| **换手率** | 因子排序的稳定性 | 越低交易成本越友好 |
| **因子半衰期** | IC 衰减到一半的时间 | 决定调仓频率 |

> 工具首选 **alphalens**（原 Quantopian 出品）：一行代码出 IC / 分组收益 / 换手率全套分析图。

### 2.4 因子处理流水线（必做）

```
原始因子
  │
  ├─ 1. 去极值：MAD 法 / 3σ 截断
  ├─ 2. 标准化：z-score（按截面）
  ├─ 3. 中性化：对行业 + 市值做回归，取残差（剥离风格暴露）
  └─ 4. 缺失值处理：行业中位数填充
        │
        ▼
    可用于合成的纯净因子
```

**为什么中性化**：一个"动量因子"如果和市值高度相关，你赚的可能是小市值效应而不是动量。中性化让因子的贡献可解释、可归因。

### 2.5 因子合成

| 方法 | 说明 |
|---|---|
| 等权合成 | 简单稳健，baseline |
| IC 加权 | 按 IR 给权重 |
| PCA / 因子模型 | 降维，去冗余 |
| **树模型（LightGBM）** | **主力方法**：把因子当特征，下期收益当标签，让模型学非线性合成 |

### 🎯 阶段二项目

1. 复现 **Alpha101 中的 5–10 个因子**；
2. 用 **alphalens** 出完整评价报告（IC / IR / 分组 / 换手）；
3. 完成一套中性化流水线（行业 + 市值），对比中性化前后 IC 变化。

**✅ 完成标志**：
- [ ] 能解释 IC / IR / 分组单调性
- [ ] 实现去极值 + 标准化 + 行业市值中性化
- [ ] 有至少 3 个 |IC| > 0.03 的有效因子

## 阶段三：金融统计与机器学习

### 核心目标

这是你的**主战场**，但金融 ML 和你熟悉的互联网 ML 有本质差异：**信噪比极低、数据非平稳、过拟合一抓一大把**。这一阶段最难的不是上模型，而是**别被模型骗了**。

### 3.1 金融时序的特殊统计性质

- **非正态、肥尾**：收益率分布峰度远大于 3，极端事件频率远高于正态假设；
- **波动率聚集**：大涨大跌后波动率自相关，方差时变（GARCH）；
- **非平稳 / 分布漂移**：训练集和测试集分布不同（regime shift），这是 ML 在金融翻车的头号原因；
- **低信噪比**：相比 CV / NLP，金融特征里噪声压倒信号。

### 3.2 主力模型：树模型，不是 LSTM

> ⚠️ **常见误区**：用 LSTM 预测股价。业内反复验证——单资产价格预测信噪比太低、过拟合严重，LSTM 几乎打不过 LightGBM。

| 任务 | 推荐模型 | 原因 |
|---|---|---|
| 横截面选股 | **LightGBM / XGBoost / CatBoost** | 处理表格特征强、可解释、抗过拟合 |
| 截面排序 | LambdaMART / 排序学习 | 直接优化排序 |
| 跨资产关系 | GNN（图神经网络） | 把股票当节点、行业/供应链当边 |
| 时序（慎用）| Transformer | 数据量足够时，但需严防过拟合 |
| 组合执行 / 仓位 | 强化学习 | 比预测价格更合适的应用 |

**标签工程**：金融里标签怎么定义比换模型重要得多。预测"未来 5 日收益率"的连续回归 vs "未来 5 日是否跑赢中位数"的二分类，效果天差地别。

### 3.3 金融 ML 的坑（必读 López de Prado）

这一节的每一项都来自《Advances in Financial Machine Learning》，是算法工程师转量化的**必读书**：

| 技术 | 解决什么问题 |
|---|---|
| **Purged k-fold CV** | 标准交叉验证在时序上有信息泄漏，需剔除标签重叠期并加 embargo |
| **组合自相关（CSS）** | 标签间相关性会虚高样本量，需用有效样本数 |
| **PBO（回测过拟合概率）** | 量化回测过拟合的严重程度 |
| **CSCV（Combinatorially Symmetric CV）** | 计算 PBO 的方法 |
| **Meta-Labeling（元标签）** | 先有预测方向，再用 ML 决定**下多大注**（二分类叠加）|
| **Fractional Differentiation** | 让序列既平稳又保留记忆，优于简单一阶差分 |

> 这些方法是把你的 ML 严谨性迁移到金融的**关键桥梁**，也是面试高频考点。

### 🎯 阶段三项目

用 **qlib（微软亚洲研究院 MSRA）+ LightGBM** 跑通 **Alpha158** 完整 pipeline，复现官方 benchmark，并：
- 实现 **Purged k-fold** 替换默认 CV，对比过拟合差异；
- 用 **Meta-Labeling** 给原模型加一层仓位决策。

**✅ 完成标志**：
- [ ] 跑通 qlib Alpha158 + LightGBM，达成接近官方的 IC
- [ ] 实现 Purged k-fold，理解为何标准 CV 会泄漏
- [ ] 能讲清楚"为什么不该用 LSTM 直接预测价格"

## 阶段四：投资组合优化

### 核心目标

从"选出好股票"到"决定每只买多少"。这是你**凸优化功底**最该发力的地方，也是大多数自学量化的人跳过、但机构极其看重的环节。

### 4.1 从马科维茨说起

均值-方差模型（MVO）：给定收益和风险，求最优权重。但它**对输入极度敏感**（协方差矩阵估计噪声大），实操直接用会翻车。

### 4.2 风险模型：协方差怎么估

| 方法 | 说明 |
|---|---|
| 样本协方差 | 噪声大，维度 > 样本时奇异 |
| **收缩估计（Ledoit-Wolf）** | 向单位阵 / 单因子模型收缩，工业标配 |
| **因子模型（Barra）** | 用风格因子解释协方差，降维且稳定 |
| DCC-GARCH | 时变条件相关（进阶）|

### 4.3 优化目标家族

| 目标 | 说明 |
|---|---|
| 最大夏普 / 最大 IR | 主流 |
| 风险平价（Risk Parity） | 各资产风险贡献相等 |
| Black-Litterman | 结合市场均衡 + 主观观点，比 MVO 稳健 |
| 最小方差 | 不预测收益，只控制风险 |
| 最大分散化 | 最大化组合分散度 |

### 4.4 带约束的凸优化（CVXPY 实操）

真实组合优化是带约束的 QP：

```
maximize     αᵀ w - λ wᵀ Σ w          ← 目标：超额收益 - 风险惩罚
subject to   1ᵀ w = 1                  ← 满仓
             0 ≤ w ≤ w_max             ← 单股权重上限（如 5%）
             |行业暴露| ≤ L            ← 行业偏离约束
             |w - w_prev|· turnover ≤ T ← 换手约束
             |风格因子暴露| ≤ F        ← Barra 风险约束
```

工具：**CVXPY**（Python 凸优化 DSL），直接把上面这套写成代码求解。

### 🎯 阶段四项目

用 **CVXPY** 写一个生产级组合优化器：输入因子预期收益 + Ledoit-Wolf 协方差，求解带"个股上限 / 行业约束 / 换手约束 / Barra 风格暴露"的 QP，输出目标持仓。

**✅ 完成标志**：
- [ ] 实现 Ledoit-Wolf 协方差收缩
- [ ] CVXPY 求解带 4 类约束的 QP
- [ ] 对比"无约束 MVO"与"带约束"的换手率和稳定性

## 阶段五：Alpha 对冲与市场中性

### 核心目标

理解 A 股机构量化的**标准产品形态**：多头股票池 + 空头股指期货 = 市场中性，赚 Alpha、不承担市场涨跌。

### 5.1 为什么要对冲

```
全多头组合收益 = Beta×市场收益 + Alpha
                    └─承担市场风险┘  └─真正的能力┘

对冲后：买入一篮子股票（Alpha 多）+ 做空股指期货（剥离 Beta）
       ≈ 纯 Alpha，与大盘涨跌基本无关 → 净值曲线平滑
```

### 5.2 股指期货

| 合约 | 标的 | 用途 |
|---|---|---|
| IF | 沪深 300 | 对冲大盘股池 |
| IC | 中证 500 | 对冲中盘股池（最常用）|
| IM | 中证 1000 | 对冲小盘股池 |

关键概念：
- **基差**：现货价 - 期货价，长期贴水（期货 < 现货）= 对冲有**隐性成本**；
- **展期（roll）**：合约到期需换月，产生展期成本；
- **保证金**：期货是保证金交易，需预留现金防止爆仓。

> A 股长期贴水意味着**纯多头 Alpha + IC 空头对冲**要持续付出基差成本，这是市场中性策略的重要现实约束。

### 🎯 阶段五项目

构建一个完整 **Alpha 对冲回测**：多头池来自阶段三的 LightGBM 选股，空头用 IC 股指期货对冲，回测净值曲线 + 计入基差 / 展期成本后的真实 Alpha、IR、最大回撤。

**✅ 完成标志**：
- [ ] 解释基差 / 展期 / 保证金
- [ ] 跑通"多头 + IC 空头"对冲回测
- [ ] 对比对冲前后的 Beta 暴露与净值波动

## 阶段六：严谨回测与风控

### 核心目标

回测是量化**最容易自欺**的环节。目标是建立一套让你"不被自己骗到"的评估体系。

### 6.1 回测引擎：事件驱动 vs 向量化

| 类型 | 速度 | 严谨度 | 适用 |
|---|---|---|---|
| 向量化（vectorbt 等） | 极快 | 中 | 快速初筛 |
| 事件驱动（backtrader / vnpy / 自研）| 慢 | 高 | 终版回测，可模拟成交细节 |

> 工业级回测通常两者结合：向量化做因子/策略初筛，事件驱动做终版验证。

### 6.2 回测陷阱（强化版）

| 陷阱 | 后果 |
|---|---|
| 未来函数 / 前视偏差 | 虚高收益 |
| 幸存者偏差 | 高估收益（漏掉退市股）|
| 数据窥探（反复试参）| 过拟合，样本外崩溃 |
| **时点错位**（财报非 PIT）| 虚假 Alpha |
| 忽略成本 | 纸上富贵 |
| 忽略涨跌停可成交性 | 高估换手执行 |
| 标签重叠 + 普通 CV | 虚高样本量（见阶段三 PBO）|

### 6.3 回测指标

```
收益类：总收益、年化收益、超额收益（相对基准）
风险类：年化波动 = 日收益σ×√252、最大回撤、下行偏差
风险调整：
   夏普 = (年化收益 - 无风险) / 年化波动
   索提诺 = (年化收益 - 无风险) / 下行偏差（分母只对低于 MAR 的偏差）
   卡尔玛 = 年化收益 / 最大回撤
交易类：胜率、盈亏比、年换手率、容量（capacity）
```

经验值：年化夏普 > 1 不错，> 2 优秀；最大回撤 < 15% 可接受。**但任何回测夏普 > 3 的策略，先怀疑过拟合**，而不是高兴。

### 6.4 样本外与 Walk-Forward

```
滚动窗口：训练 → 验证 → 测试 → 向前滚动
强调：永远不要用全样本一次性选参；参数越少越好（3–5 个）；
      做参数稳定性检验（最优解附近性能是否平滑）。
```

### 6.5 风控体系

**仓位**：
- 凯利公式 `f* = (bp - q) / b`（p 胜率、q=1-p、b 盈亏比）理论上给出最优注码，但**实际量化几乎不用满仓 Kelly**（会高杠杆），普遍用 **Fractional Kelly**（如 0.25×Kelly）+ 波动率目标。

**组合约束**（呼应阶段四）：单票上限、行业集中度、总仓位、相关性控制。

**止损**：固定 / ATR / 时间 / 追踪止损——量化策略里止损更多是风控阀值，而非策略核心。

### 🎯 阶段六项目

实现一个**带过拟合检测的回测引擎**：支持事件驱动成交、成本与涨跌停约束、Walk-Forward，并用 **CSCV 计算 PBO**，量化你的策略有多大比例是过拟合。

**✅ 完成标志**：
- [ ] 解释至少 5 种回测陷阱及规避方法
- [ ] 实现带约束的事件驱动回测
- [ ] 算出策略的 PBO，并理解其含义

## 阶段七：实盘交易

### 核心目标

从模拟到实盘，建立稳定的交易系统。**对个人研究者，模拟盘阶段远比想象的重要**——很多回测漂亮的策略一上模拟盘就原形毕露。

### 7.1 交易系统架构

```
数据层（行情/财务/另类）
   │
策略层（信号生成 + 组合优化）
   │
执行层（订单路由 / 母单拆分 / 算法交易）
   │
风控层（事前/事中/事后）
   │
监控层（日志/告警/复盘）
```

### 7.2 对接方式

| 方式 | 说明 |
|---|---|
| 模拟盘 | 券商模拟，验证实时表现 |
| 券商 API / 框架 | QMT、恒生 PTrade、**vnpy**（开源，主流）|
| CTP | 期货专用 |
| 交易所 API | 加密货币 |

### 7.3 个人量化的现实（重要）

坦诚讲，个人做量化面对**数据、资金、速度三重劣势**，因此：
- **避开高频 / 微秒级竞争**——那是军备竞赛，个人无胜算；
- **聚焦中低频**（周 / 月调仓）：数据需求小、容量大、对速度不敏感，是个人能做出相对优势的区间；
- **小资金验证**：模拟 → 小资金实盘 → 逐步加仓，永远先保本金。

### 🎯 阶段七项目

把你的策略上 **vnpy 模拟盘**跑 1 个月，做完整盘前 / 盘中 / 盘后流程，写复盘报告：实盘与回测的偏差来源。

**✅ 完成标志**：
- [ ] 策略上模拟盘稳定运行
- [ ] 有完整复盘流程
- [ ] 能指出回测与实盘的 3 个主要偏差

## 附录

### A. 速查：传统技术指标（降级，仅供查阅）

> 现代量化很少直接用这些"民间派"指标，了解其逻辑即可，不建议作为策略核心。需要时查阅：

```
SMA = sum(close, n) / n
EMA = α·close + (1-α)·EMA_prev,   α = 2/(n+1)

MACD   = EMA(12) - EMA(26)
Signal = EMA(MACD, 9)
RSI    = 100 - 100/(1 + 平均涨幅/平均跌幅)

Boll_upper = MA(20) + 2·STD(20)
Boll_lower = MA(20) - 2·STD(20)

ATR = MA(TrueRange, 14)
TrueRange = max(H-L, |H-C_prev|, |L-C_prev|)
```

### B. 关键指标公式速查

```
总收益率 = (期末净值 - 1) × 100%
年化收益 = (期末净值^(252/交易日) - 1) × 100%
最大回撤 = max((峰值 - 当前) / 峰值)
夏普比率 = (年化收益 - 无风险利率) / 年化波动率
索提诺   = (年化收益 - 无风险利率) / 下行偏差
IC       = corr(因子截面值, 下期截面收益)        # 常用秩相关 Rank IC
IR       = mean(IC) / std(IC)
胜率     = 盈利次数 / 总交易次数
盈亏比   = 平均盈利 / 平均亏损
```

### D. 术语表

|---|---|

## 心态建议

- **量化是低信噪比的长跑**，不是一夜暴富——任何回测夏普 > 3 先怀疑过拟合；
- **保住本金比赚钱重要**，Fractional Kelly 而非满仓；
- **策略会衰减**（Alpha decay），今天的圣杯明天就是垃圾，需要持续进化；
- **纪律 > 策略**：能稳定执行平庸策略，胜过摇摆执行神策略；
- **个人聚焦中低频**，避开机构军备竞赛的高频战场。
